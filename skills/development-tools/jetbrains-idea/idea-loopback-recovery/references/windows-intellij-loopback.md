# Windows 环境下的 IntelliJ IDEA 回环交互说明

## 背景

在 Windows 环境中，IntelliJ IDEA 处理部分 Git 登录、授权、外部浏览器回调时，可能会依赖本地回环地址、系统默认浏览器、临时端口监听或中间凭据桥接。

一旦其中某个环节状态异常，就可能出现：

- 浏览器已经打开，但授权无法完成。
- 回调地址无法正常返回到 IDEA。
- IDE 中的操作长期等待，没有明确失败提示。
- 同一个动作在命令行和 IDE 中表现不一致。

## 典型判断思路

可以先按以下顺序判断：

1. 问题是不是只发生在 IDEA 内。
2. 浏览器是否真的完成了授权。
3. 回调地址是否存在本地白页、拒绝访问或直接卡死。
4. IDEA 进程是否存在残留状态，没有完全释放。

## 为什么先做低风险恢复

这类问题常见于临时状态错乱，而不是配置被永久破坏，所以优先执行：

- 彻底退出 IDEA。
- 清理残留进程。
- 清理锁定痕迹。
- 再次触发问题验证。

这样做的优点是影响面小、回滚简单，也便于快速确认问题是否只是临时故障。

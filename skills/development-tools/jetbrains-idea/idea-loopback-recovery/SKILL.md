# 适用场景

当 IntelliJ IDEA 中的 Git 操作出现授权页无法打开、登录无法继续、本地回环地址异常、askpass 失效、操作卡死或反复失败时，使用本 Skill。

# 排查目标

- 判断问题是否集中在 IDEA 内部 Git 交互链路。
- 确认是不是本地回环地址或 askpass 触发了阻塞。
- 尽量用最小影响方式恢复 IDEA 的 Git 可用性。
- 必要时清理缓存、锁文件或临时状态。

# 处理步骤

## 第 1 步：记录当前现象

先确认这些信息：

- 具体是哪一个 Git 操作失败，例如 `push`、`pull`、登录、拉起授权。
- 是否只在 IDEA 内失败，而命令行 Git 正常。
- 是否出现本地网页打不开、白页、回调失败、浏览器不响应等现象。
- 是否近期升级过 IDEA、插件、Git 或系统代理配置。

如果命令行 Git 也异常，说明问题可能不只在 IDEA，应先排查全局 Git 配置。

## 第 2 步：先做低风险恢复

优先尝试以下低风险动作：

1. 完全退出 IntelliJ IDEA。
2. 打开任务管理器，确认相关 Java / JetBrains 进程已经结束。
3. 重新打开 IDEA，复现一次问题。
4. 如果问题仍在，再确认命令行 Git 是否正常。

这个步骤的目的，是先排除“IDE 进程未完全退出导致状态残留”的情况。

## 第 3 步：判断是否是回环 / askpass 问题

如果出现以下特征，优先怀疑本地交互链路异常：

- IDE 里等待浏览器授权，但浏览器没有正常完成。
- 打开的地址类似本地回调地址，但页面无响应。
- Git 操作没有明确报错，只是一直卡住。
- IDEA 内置认证流程和外部浏览器之间状态不同步。

这类问题通常不一定是 Git 本体损坏，而是“IDEA → 本地授权流程 → 回调 → 凭据回写”这条链路中某一环卡住。

## 第 4 步：清理可能的残留状态

如果确认问题高概率发生在 IDEA 侧，可以执行以下动作：

- 关闭 IDEA。
- 运行 `scripts/clear-idea-system-lock.ps1`。
- 重新启动 IDEA。
- 再次尝试触发 Git 操作。

这个脚本的目标是清理常见的系统锁定痕迹与临时残留，降低旧状态继续干扰的概率。

## 第 5 步：验证恢复结果

至少验证以下两项：

- 在 IDEA 内执行一次原先失败的 Git 操作。
- 确认不再出现卡死、授权无法完成或本地回环无响应的问题。

如果命令行 Git 正常，IDEA 内也恢复正常，可以认为这次恢复有效。

# 如果没有恢复

按下面顺序继续细化排查：

1. 对比命令行 Git 与 IDEA 内 Git 的行为差异。
2. 检查 IDEA 的 Git 可执行文件指向是否正确。
3. 检查系统代理、VPN、浏览器默认协议处理是否近期发生变化。
4. 回看是否是某次 IDEA 或插件升级后才出现。
5. 必要时再考虑清理更深层的缓存目录或重置相关设置。

# 使用建议

- 优先把这个 Skill 当成“排查流程”而不是一次性脚本。
- 每次处理时记录现象、动作和结果，便于后续继续沉淀。
- 如果你以后整理出更多 IDE 类问题，可以沿用这个结构继续扩展。
